Set集合

一、介绍

Set 是无序集合的顶层接口。

特点：

​    1.元素无序(存入与取出的顺序不一样)

​    2.元素不可通过索引遍历

​    3.可以不能重复

常用子类：

​   HashSet

​   TreeSet

​   LinkedHashSet

二、详解

1.HashSet集合

​ （1）特点：

​    底层使用哈希表

​ （2）常见问题

​    HashSet是线程不安全的，效率高

​     HashSet初始容量为16，超过容量的0.75倍时，容量扩大2倍。即：当第一次超出12时，自动扩容至32

​ （3）使用

​    遍历方式

Code

​ HashSet元素唯一性解释：依赖hashCode()和equals()方法

​ 判断两个对象(集合中已经存在的某个元素 和 要添加的元素)的哈希值是否相同.

​ 哈希值相同: 说明可能是同一个元素, 程序继续往下执行.

​ 哈希值不同: 说明不是同一个元素, 就添加.

​ 比较两个对象的地址值是否相同

​ 地址值相同: 说明是同一个元素, 不添加.

​ 地址值不同: 说明可能不是同一个对象, 程序继续往下执行.

​ 判断要添加的元素是否为null

​ 如果为null:不是同一个元素, 就添加.

​ 如果不为null: 说明可能是同一个元素, 程序继续往下执行.

​ 比较对象的各个属性值是否相同

​ 如果相同: 说明是同一个元素, 不添加.

​ 如果不同: 说明不是同一个元素, 就添加.

​ 综上所述，如果自定义类不重写hashCode()和equals()方法，将不去重

2.TreeSet集合

​ （1）特点：

​       底层使用红黑树（二叉排序树）

​ （2）常见问题

​       TreeSet是线程不安全的，效率高

​ （3）使用

​       保证元素有序

​       自定义类中实现Comparable接口，并重写CompareTo(E e)方法

​       使用TreeSet(Comparator comparator)构造器

3.LinkedHashSet集合

​ （1）特点：

​       底层使用哈希表+数组

​       有序

​ （2）常见问题

​       LinkedHashSet是线程不安全的，效率高
