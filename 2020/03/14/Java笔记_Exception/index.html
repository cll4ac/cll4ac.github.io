Exception

一、概述

​ 异常是一种现象，表示阻止程序正常执行的错误或者情况。

二、体系

三、解释

​ 1、Throwable：Java 语言中所有错误或异常的超类。

​ 2、rror：由虚拟机抛出的，表示内部系统错误，用户不能修复的问题。

​ 3、Exception：由程序和外部环境所引起的错误，这些错误能被程序捕获和处理。其又分为两种类别

​ 1、运行时异常(runtime exception)：使用RuntimeException类表示的，它描述的是程序设计错误，运行时，常由Java虚拟机抛出的。（可处理可不处理）

​ 2、编译时异常：除了RuntimeException、Error以及它们的子类，其它的异常都是编译时异常，它描述的是编译器会强制程序员检查并通过try-catch块处理，或者在方法头出声明。（必须处理）

四、使用

​ 1、声明异常：

   作用：告诉方法调用者，使用该方法可能会遇到什么样的异常。

   如何使用：当存在编译时的异常且不想在方法内处理，则必须声明异常。

   格式：在方法声明处声明异常。

java

​ 注意事项：

    如果父类方法声明异常，子类继承父类方法并重写时，可声明异常，可不声明，如果要声明异常，这个异常不能比父类的异常大。如果父类方法没有什么异常，则子类也不能声明异常。

    如果父类构造方法声明编译时异常，则子类构造必须声明该异常。若父类构造声明运行时异常，子类构造可声明可不声明。

​ 2、抛出异常：

   作用：检测到错误的程序可以创建一个合适的异常类型实例并抛出它。例如，创建一个方法，定义这个方法的参数必须为负数，但检测到调用者传入一个正数，这时就可以抛出IllegalArgumentException异常，并且加上提示信息，让调用者知道犯下的错误。

   如果使用：想抛出一个异常并附加提示信息时使用。

   格式：

   作用：捕获异常并处理它。

   何时使用：当方法不能抛出异常，且有编译时异常时，就必须捕获异常，并处理它。或者自己想要处理异常。

   格式：

java

      1.try{}后面必须有catch(){}或finally{}

      2.finally必须写在catch后面
