IO流

1、概述

​  1、文件分类：文件可以分为文本文件或二进制文件。

​    文本文件：如果能用文本编辑器，比如Windows下的记事本或者LINUX下的vi编辑器，进行处理的文件。

​    二进制文件：不能使用文本编辑器来读取的文件。这些文件是给程序来读取的。

​  2、IO流分类：IO流分为 字符IO 或 字节IO

​    字节IO：用来传输二进制文件，文本文件在计算机中也是以二进制传输的，所以字节IO也可以传输文本文件。

​    字符IO：用来传输文本文件。其本质就是：字节IO+编码表

​  3、编码表分类：将字符以某种规则，将字符存入到计算的过程，即为编码。其分类如下：

​    ASCII编码：每个字符占一个字节，只能表示128个字符。原来码表一不满足需求，后又继续扩展。

​    GBxx字符集:

​       GB2312:最早的中文码表，兼容ASCII。每个字占两个字节，共收录6700多个汉字。

​      GBK:是GB2312的升级，融合了更多的中文汉字，其在国内经常使用，收录2万多个汉字。

​      GB18030:GBK的升级，收录了更多的中文汉字。每个字占4个字节，共收录7万多个汉字

​   Unicode编码:

​      utf-8:为了统一世界语言而面世的编码，兼容ASCII,每个字符用1-4个字节表示，当字符在ASCII码表范围，就用一个字节表示。汉字在表中占3个字节。

​      ISO-8859-1：欧洲码表，用1个字节表示数据，兼容ASCII编码 4、解码：文本文件是提供给人看的，但是符号文字都是通过编码后，以二进制方式存入计算机，怎么样才能看到呢？那就是需要通过解码的方式将存入的二进制还原成符号文字。即某种规则将存储在计算机中的二进制解析成符号文字。注意：每种编码方式都不一样，所以一般编码和解码的规则要一样。

 7、总结

​   1、普通的字节流使用一次一个字节的传输方式，最慢。

​   2、缓冲的字节流使用一次一个字节的传输方式，慢。

​   3、缓冲的字节流和普通的字节流使用一次一个字节数组的传输方式，都快。但是普通的字节流需要创建容量较大的字节数据传输才比较快，这样会消耗资源，所以一般会使用缓冲流的一次一个字节数组传输。

​   4、缓冲的字节流在写数据之后，如果不调用close()方法，需要调用flush()方法，将缓冲的数据写入文件。调用close()方法就不需要调用flush()方法，因为close()内部会自动调用flush()方法。close()方法是将流关闭，之后再调用write()方法时，只会写入到缓冲区内，不能写到文件中去。
